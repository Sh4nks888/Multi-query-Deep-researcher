# 任务 3: 代表系统

## 任务概览
- 任务主题：探索多模态Agent
- 任务名称：代表系统
- 任务目标：调研当前具有代表性的多模态Agent项目、论文与产品，比较其输入模态、任务能力、开源状态和应用场景。
- 检索查询：multimodal agents examples GPT-4o Gemini robotics GUI agent papers

## 来源概览
1. Medium《From Multimodal Agents to Moving Pictures》：概述多模态Agent从感知建模、跨模态注意力、联合嵌入到GPT-4o、Gemini、Sora等系统的演进，并列出Perceiver、Flamingo、PaLI、GPT-4、Gemini 1.5等关键研究。
2. Berkeley RDI《Multimodal Agent》课件：强调从感知到行动的VLA-M范式，覆盖Web、桌面、移动、物理机器人等Agent类别，并重点介绍OSWorld执行式评测环境。
3. AI Minds《GPT-4o Complete Review...》：指出GPT-4o发布后多模态AI Agent热度上升，并连接语音、视觉、IoT、对话Agent等产业方向。
4. Cobus Greyling《Multi-Modal Agentic Applications》：从工程角度说明Agent由基础模型、任务分解、工具调用、人类介入组成，多模态能力让Agent能处理图像、GUI元素和开放工具链。
5. OSU-NLP GUI-Agents-Paper-List：收录542篇GUI Agent相关论文，覆盖Web、Desktop、Mobile、General GUI，以及benchmark、dataset、framework、GUI grounding、safety、security等关键词。

## 关键发现
1. 代表性多模态Agent可分为通用原生多模态模型、GUI/Computer-use Agent、机器人/物理Agent、工程框架型Agent四类。GPT-4o、Gemini偏通用入口；OSWorld/WebArena/Mind2Web等代表计算机操作评测生态；Figure Helix等代表物理行动方向；LangChain + GPT-4o-mini代表可落地的应用开发路径。
2. 多模态Agent的核心能力不只是“支持多种输入”，而是跨模态表示、对齐、推理与行动闭环。Perceiver、Flamingo、PaLI、Gemini等研究路线体现了从多输入感知到统一推理架构的演进。
3. GUI Agent是当前最活跃、最接近生产力工具的分支。OSU-NLP列表收录542篇GUI Agent论文，按环境统计Web 218、Desktop 124、Mobile 163、General GUI 114；按主题统计benchmark 177、dataset 101、framework 61、reinforcement learning 57，说明该领域已从演示走向系统化评测、数据构建和训练方法竞争。
4. 评测正在从静态问答转向真实环境执行。OSWorld包含369个真实世界电脑任务，覆盖桌面应用、多应用工作流、文件I/O，并通过虚拟机初始状态和执行式评测脚本验证最终状态，解决过去Mind2Web/WebArena等环境任务范围或交互真实性不足的问题。
5. 当前短板集中在可靠行动、长程规划、安全、成本和开放性。Berkeley课件指出LLM/VLM距离真实数字Agent仍有明显差距；截图、a11y tree、Set-of-Marks等观测方式效果因模型而异。闭源系统能力强但可控性和复现差，开源系统便于研究但整体性能与工程成熟度仍落后。

## 代表系统对比摘要
- GPT-4o：输入模态覆盖文本、图像、音频/语音，偏实时对话、视觉理解、助手型交互；闭源产品；应用于客服、教育、办公、语音视觉助手。
- Gemini 1.5/2.5：文本、图像、音频、视频、长上下文输入，强调原生多模态、长上下文、深度推理和编码；闭源/产品API；应用于Google生态、研究分析、代码、视频/文档理解。
- OSWorld + GPT-4V/Gemini/Claude/CogAgent等：输入包括任务指令、截图、a11y tree、历史动作，输出键鼠/程序化动作；OSWorld为开源/研究基准，模型可闭源或开源；应用于桌面自动化、多应用办公任务。
- WebArena/Mind2Web/BrowserGym/WebVoyager：主要处理网页截图、DOM/可访问性树、文本指令和浏览器动作；多为研究基准或开源项目；应用于网页自动化、信息检索、在线服务操作。
- LangChain + GPT-4o-mini：输入可包含文本和图像，能力取决于工具集合；框架开源、模型API多为闭源；适用于快速构建多模态检索、图像问答、工具调用工作流。
- Figure Helix/机器人VLA方向：输入包含视觉、语言和机器人传感/动作空间，输出物理动作；多为闭源产品或研究系统；应用于人形机器人、仓储、家庭/工业操作。
