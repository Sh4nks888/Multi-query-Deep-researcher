# 任务 3: 技术栈选型

## 任务概览
- 任务主题：Agent学习路线
- 任务名称：技术栈选型
- 任务目标：调研主流Agent开发框架、平台和生态工具，比较其适用场景、学习成本与社区活跃度。
- 覆盖范围：LangChain/LangGraph、LlamaIndex、AutoGen/AG2、CrewAI、OpenAI Agents SDK，并扩展到 Claude Agent SDK、Semantic Kernel、Pydantic AI、MCP、观测、评测、安全、向量数据库等生态工具。

## 来源概览
- PE Collective《AI Agent Frameworks: LangGraph vs CrewAI vs AutoGen 2026》：强调 LangGraph、CrewAI、AutoGen 三类主流范式；提到 2026 年 AutoGen 1.0 GA、LangGraph 0.3.x、CrewAI 0.95、OpenAI Agents SDK planning module 等演进。
- Alice Labs《Best AI Agent Frameworks 2026》：基于 2024–2026 年 18+ 次生产部署，给出 LangGraph、Claude Agent SDK、CrewAI、AutoGen/AG2、Semantic Kernel、LlamaIndex、Pydantic AI 的生产排序和约束选择法。
- Reddit r/LangChain 生态清单：覆盖 260+ Agent 资源，展示框架之外的观测、评测、协议、向量数据库、安全工具已成为 Agent 工程必需品。
- Turing《A Detailed Comparison of Top 6 AI Agent Frameworks in 2026》：覆盖 LangGraph、LlamaIndex、CrewAI、Semantic Kernel、AutoGen、OpenAI Swarm 等，适合作为横向对照材料。
- YouTube《OpenAI SDK vs CrewAI vs LangGraph vs AutoGen》：面向初学者和自动化工程师，强调 OpenAI SDK、CrewAI、LangGraph、AutoGen 的入门认知和场景选择。

## 关键发现
1. 技术栈选型应从约束出发，而不是追逐单一“最佳框架”。LangGraph 偏复杂可控编排，LlamaIndex 偏数据/RAG，CrewAI 偏角色协作，AutoGen/AG2 偏对话式多 Agent 研究，OpenAI Agents SDK 偏 OpenAI 平台原生能力。
2. LangGraph 是生产级复杂 Agent 编排的强势默认选项。其状态图模型适合分支、循环、重试、checkpoint、人类介入和观测，但学习曲线较陡，简单任务会显得样板代码偏多。
3. CrewAI 与 AutoGen/AG2 代表两种多 Agent 路线：CrewAI 是“组织分工式”，适合研究-写作-审阅等流程型协作；AutoGen/AG2 是“对话涌现式”，适合研究实验、代码执行和多智能体问题求解。
4. LlamaIndex 的价值在数据层 Agent。它不是最强通用编排器，而是更适合私有知识库、索引、检索、query engine、RAG-grounded agents 等数据密集型任务。
5. Agent 学习路线必须覆盖框架外生态。观测工具 LangSmith/Langfuse/Arize Phoenix/Helicone，协议 MCP/A2A/Function Calling，评测 SWE-bench/GAIA/WebArena/AgentBench，安全 Guardrails AI/NeMo Guardrails/LLM Guard，向量数据库 Chroma/Qdrant/Milvus/Weaviate/Pinecone 都应纳入工程闭环。

## 选型建议
- 入门阶段：优先用 CrewAI 或 OpenAI Agents SDK 做小型自动化 demo，建立 tool calling、memory、multi-step execution 的直觉。
- 数据/RAG 阶段：学习 LlamaIndex，掌握文档加载、切分、索引、检索、重排、查询路由和私有知识库 Agent。
- 生产编排阶段：学习 LangGraph，重点掌握状态图、checkpoint、streaming、人类介入、失败恢复和可观测性。
- 多 Agent 研究阶段：学习 AutoGen/AG2，理解 group chat、code execution、agent conversation 与实验型协作模式。
- 企业落地阶段：按组织技术栈补充 Semantic Kernel（Microsoft/.NET）、Pydantic AI（类型安全 Python）、Claude Agent SDK 或 OpenAI Agents SDK（模型厂商原生能力）。
