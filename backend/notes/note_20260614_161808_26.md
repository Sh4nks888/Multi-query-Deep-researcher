# 任务 1: 概念边界

## 任务概览
- 任务主题：探索多模态Agent
- 任务名称：概念边界
- 任务目标：厘清多模态Agent的定义、核心能力、与传统LLM Agent及多模态模型的区别，建立后续调研的统一框架。
- 检索查询：multimodal agent definition capabilities LLM agent difference survey

## 来源概览
1. Towards Rationality in Language and Multimodal Agents: A Survey，NAACL 2025 Long Paper。该文从“理性智能体”角度定义 agent：能够通过自然语言或视觉、音频、代码等多模态输入感知并理解环境，并为特定目标采取行动；系统形态可从单一LLM、多模态基础模型，到集成多智能体、工具、知识库、符号推理模块的复杂系统。
2. Large multimodal agents: a survey，Visual Intelligence / Springer Nature。该文将 LLM-driven multimodal agents 称为 Large Multimodal Agents（LMAs），强调它们是 LLM Agent 向多模态域的扩展，用于解释并响应多模态用户查询，处理更复杂、更细粒度的任务；并指出评测方法分散是当前关键挑战。
3. Large Multimodal Agents vs. Multi-agent Systems: A Comparative Analysis，Medium。该文从工程架构角度区分 LMA 与多智能体系统：LMA 偏向增强单个智能体对文本、图像、音频、视频等多模态信息的处理与行动能力；多智能体系统偏向多个异构智能体之间的协作、分工与协调。
4. Large Language Model Agents: A Comprehensive Survey on ...。该来源给出 LLM Agent 的基础定义：以大语言模型作为核心推理引擎，感知环境、做出决策并执行行动。

## 关键发现
1. 多模态Agent的概念边界不应等同于“能看图的LLM”，而应定义为“以目标驱动、能感知多源模态并据此规划、决策、行动的智能体系统”。其核心在于 agentic loop：感知环境、形成状态、推理规划、调用工具或执行动作、接收反馈并更新策略。
2. 与传统LLM Agent相比，多模态Agent的根本扩展是感知与环境 grounding。传统LLM Agent主要依赖文本输入和参数知识，容易受限于封闭知识、幻觉与不稳定输出；多模态Agent通过视觉、音频、视频、代码、外部工具与知识源引入现实证据，有助于提升信息扎根性与可靠性。
3. 与多模态模型相比，多模态Agent的差异在“行动闭环”而非“模态能力”。多模态模型侧重跨模态理解/生成，如CLIP、BLIP-2、LLaVA、GPT-4V、GPT-4o、Gemini等；多模态Agent则在这些模型之上加入目标、记忆、规划、工具使用、执行反馈与任务完成机制。
4. 多模态Agent与多智能体系统是两条可组合但不同的维度。前者回答“单个或核心智能体能处理多少模态、如何理解现实环境”；后者回答“多个智能体如何分工协作”。复杂系统可以同时是多模态Agent系统和多智能体系统。
5. 后续调研可采用统一框架：定义层关注 agentic loop，能力层关注感知、融合、推理、规划、记忆、工具调用、行动与反馈，系统层关注单体/多体、模型/工具/知识库集成，评测层关注任务成功率、事实扎根、逻辑一致性、安全性、鲁棒性与跨模态泛化。

## 价值判断
- 概念上，多模态Agent是从“语言中心智能体”走向“环境中心智能体”的过渡形态。
- 工程上，它不是单一模型能力升级，而是模型、工具、记忆、环境接口和反馈控制的系统工程。
- 研究上，当前领域仍缺少统一评测基准，特别是对跨模态任务完成、长期规划、现实反馈和理性决策质量的可比性评价。
