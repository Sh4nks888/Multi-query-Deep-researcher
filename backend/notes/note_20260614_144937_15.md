# 任务 1: 技术架构

## 任务概览
- 任务主题：探索 Agent 应用技术
- 任务名称：技术架构
- 任务目标：梳理 Agent 应用的核心技术栈与典型架构，明确 LLM、工具调用、记忆、规划、执行与反馈机制如何协同。
- 检索查询：Agent 应用 技术架构 LLM 工具调用 记忆 规划 执行框架

## 来源概览
- CSDN / AI Agent 技术社区《【大模型LLM】Agent 架构图解》：将 LLM Agent 拆解为控制中枢、感知与行动、记忆系统、安全控制，并对比单智能体、多智能体、符号-神经混合、强化学习驱动等架构。
- 飞书文档《AI Agent 万字长文总结：规划、工具、执行、记忆》：强调任务执行 Agent 获取记忆、执行任务并将结果回写记忆存储的流程。
- SegmentFault《从0到1构建AI Agent：架构、代码与工程实践》：将 AI Agent 定义为以 LLM 为核心推理引擎，配备感知输入、规划决策、记忆存储、工具执行四大能力模块的自主 AI 系统。
- 掘金《基于大模型的 Agent 技术框架》：提出 Planning、Memory、Tools、Action 四大要素，详述 ReAct、Function Calling、Plan-and-Execute、Self-Ask、Critique Revise 等认知框架。
- 菜鸟教程《Agent 架构》：从 Single Agent Loop 到 Plan & Execute、多 Agent 协作等架构说明 Agent 的本质是感知、推理、行动循环。

## 关键发现
1. Agent 应用的核心架构已从单次 LLM 调用演进为感知—推理—行动—反馈闭环。LLM 不再只是问答接口，而是作为 Brain / 推理中枢，结合规划、工具、记忆、执行器与反馈机制持续推进任务。
2. LLM 与工具调用形成能力分工：LLM 负责目标理解、路径选择和决策排序，工具负责搜索、计算、代码执行、数据库/API 访问、文件操作等确定性动作。工具描述、参数 schema、权限边界、错误处理和幂等性是工程落地关键。
3. 规划机制决定复杂任务的可控性。ReAct 通过 Thought → Action → Observation 循环适合动态探索；Plan-and-Execute 将规划和执行分离，适合可审查、可回滚、可治理的企业流程。
4. 记忆系统正在从上下文拼接升级为长期状态管理与知识检索基础设施，包括短期工作记忆、长期用户偏好、任务历史、向量检索、RAG 知识库和执行结果沉淀。
5. 典型架构分化为单 Agent、多 Agent、符号-神经混合、强化学习驱动等路线。单 Agent 简单易调试，多 Agent 适合复杂项目协作，符号-神经混合适合强逻辑约束任务，强化学习驱动适合动态环境交互。

## 价值判断
- Agent 架构的关键不在于“让模型更会说”，而在于构造一个能持续观察、选择、执行、校验和修正的运行系统。
- 工程实践中应优先关注上下文管理、工具治理、计划可审查性、执行安全、反馈闭环和评估监控。
- 未来 Agent 应用会从简单自动化扩展到复杂业务流程、软件工程、数据分析、智能客服和物理世界交互，但可靠性、安全性和成本控制仍是主要瓶颈。
